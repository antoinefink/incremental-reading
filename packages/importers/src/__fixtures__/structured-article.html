Building a Review Queue

A good queue balances three concerns, summarized below.

The three concerns

  • Priority — high-value material is protected first.
  • Recency — newly imported items must not dominate.
  • Stagnation — postponed items resurface deliberately.

An ordered procedure

  1. Score each candidate by priority and due date.
  2. Bury siblings so they are not shown back to back.
  3. Cap the daily budget so overload never happens.

The application succeeds when users can import too much material without drowning.

The core selection looks like this:

function next(candidates) {
  return candidates.sort(byScore)[0];
}

That is the whole loop, repeated every day.
